Summary

This project was designed to develop object-oriented programming skills, including designing, documenting, testing and debugging of Java code as part of the introduction to programming unit.

The task

The simulation would model a pinball machine and the collisions of different objects with unique effects and end once a ball passes through the gap at the bottom of the machine.

Goals

  • Create walls, objects that are black in colour that are impassable to all other objects
  • Create ball type one, features, change direction when colliding with a wall, change speed when colliding with another ball.
  • Create the bumpers, Bumpers are static circular objects within the machine, which will always be grey in colour
  • Create the exit gap to end the simulation
  • Create ball type two, feature change colour when colliding with the edge, change size when colliding with a ball.
  • Create hole object, feature, balls will be removed if the fit inside the hole, glide over it if the do not.
  • Implement scoring system
    • Colliding with a wall - 1 point
    • Colliding with a bumper - 2 points
    • Colliding with another pinball - 5 points
    • Coming into contact with a hole - lose all points collected by that pinball
  • Code quality based on: its overall structure (ensuring that the principles of object-oriented programming have been incorporated), its readability and structure, and the quality of the algorithms used.

Stretch goals

  • Display high score table at the end of the simulation
  • Documentation

Results

Each goal was met including the stretch goals resulting in an overall grade of 100% for this project upon presentation.

Reflection

Collision detection was a lot of fun in particular I enjoyed working with a brief and being set lose to make the solution as however I saw best this was the first assessed project I had worked on with such a set up and it was immensely interesting researching the many different ways to approach the core problems. I found the layout of BlueJ to be very useful for showing class inheritance and dependencies it was quite satisfying reducing the dependencies and ensuring the inheritance was a simple logical flow of increasingly complex classes and having that graphical display showing the results.
